Model Architecture

Sleep LLM features a custom transformer architecture specifically designed for temporal biosignal processing and multimodal sleep data analysis.

üß†

Foundation Transformer

Our base model uses a modified transformer architecture with hierarchical attention mechanisms optimized for long-sequence biosignals.

  • 12 billion parameters
  • 32-layer transformer
  • 1024-dimensional embeddings
  • Multi-head attention (16 heads)
üìä

Time-Series Encoders

Specialized encoders process multi-channel physiological signals with temporal convolutions and attention pooling.

  • Multi-scale temporal convolutions
  • Adaptive sampling rates
  • Signal-specific normalization
  • Artifact detection and removal
üåô

Context Integration

Contextual information including medications, environmental factors, and lifestyle data is integrated through cross-attention mechanisms.

  • Cross-modal attention layers
  • Temporal context windows
  • Hierarchical feature fusion
  • Causal masking for inference
üéØ

Task-Specific Heads

Multiple specialized output heads enable simultaneous sleep staging, disorder detection, and personalized recommendation generation.

  • Multi-task learning framework
  • Shared representations
  • Task-specific fine-tuning
  • Uncertainty quantification

Python SDK

from naptick import SleepLLM

# Initialize client
client = SleepLLM(api_key="your_api_key")

# Analyze sleep data
result = client.analyze(
    psg_file="path/to/sleep_study.edf",
    tasks=["sleep_staging", "apnea_detection"],
    patient_info={
        "age": 35,
        "medications": ["melatonin"]
    }
)

print(f"Sleep efficiency: {result.sleep_efficiency}%")
print(f"REM sleep: {result.stages.rem_percentage}%")
